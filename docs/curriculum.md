### 4번. 모빌리티 정책보고서 자동 생성 (3h) ★

| 회차 | 시설 | 주요 교육내용 |
|---|---|---|
| 4-1 | 강의실 | 정책보고서 구조 설계 및 GPT 프롬프트 엔지니어링 |
| 4-2 | 실습실 | Cursor로 보고서 생성 웹 UI 및 미리보기 구현 |
| 4-3 | 실습실 | 차트·AI 이미지 포함 DOCX 원클릭 자동 생성 실습 |

#### 4-1 강의: 보고서 자동화 설계 (강의실, 1h)

- 정책보고서 구조 이해: 표지 → 요약 → 본문 → 핵심 발견 → 정책 제언 → 부록
- 데이터 → 보고서 매핑 전략 (설문 응답값 → 섹션별 분석 텍스트)
- GPT 시스템 프롬프트 설계 방법론
- `response_format: { type: "json_object" }` 활용 구조화 출력 기법

#### 4-2 실습: 보고서 생성 웹 UI 구현 (실습실, 1h)

- Cursor 바이브코딩으로 Next.js API 라우트 생성
  - `GET /api/report/[sessionId]` — 보고서 JSON 반환
- 미리보기 모달 클라이언트 컴포넌트 구현 (`ReportButton.tsx`)
- 서버 인메모리 캐시 설계: `Map<sessionId, PolicyReport>`
  - 핵심 개념: 동일 세션 재요청 시 AI 재호출 없이 즉시 반환

#### 4-3 실습: 차트·이미지 포함 DOCX 생성 (실습실, 1h)

- **QuickChart.io** API로 설문 데이터 차트 자동 생성 (설치 불필요)
  - 수치형 응답 → 가로 막대 차트
  - 범주형 응답 → 도넛 차트
- **DALL-E 3**으로 보고서 커버 이미지 AI 생성
  - 프롬프트 엔지니어링 실습 (정책 일러스트 스타일)
- **`docx` 라이브러리**로 Word 파일 빌드
  - `ImageRun`, `Table`, `Paragraph`, `HeadingLevel` 활용
- `GET /api/report/[sessionId]/docx` — 파일 스트리밍 다운로드

### 5번. 보고서 자동화 배포 및 알림 연동 (4h) ★

| 회차 | 시설 | 주요 교육내용 |
|---|---|---|
| 5-1 | 강의실 | 자동화 파이프라인 설계 및 배포 환경 이해 |
| 5-2 | 실습실 | Slack Webhook 연동 — 보고서 완성 자동 알림 구현 |
| 5-3 | 실습실 | Vercel/Render 클라우드 배포 및 실서비스 동작 확인 |
| 5-4 | 실습실 | 종합 프로젝트: 모빌리티 데이터 → 보고서 → 슬랙 알림 A to Z |

#### 5-1 강의: 자동화 파이프라인 설계 (강의실, 1h)

- 자동화 트리거 종류: 웹훅(버튼) / 스케줄(Cron) / 이벤트(DB 변경)
- 환경 변수 관리 전략 (`.env.local` → Vercel 환경 변수)
- Vercel vs Render 선택 기준
- Serverless 함수 실행 시간 제한 고려사항 (보고서 생성 최적화)

#### 5-2 실습: 슬랙 알림 연동 (실습실, 1h)

- Slack Incoming Webhook URL 발급
- `lib/server/slackNotifier.ts` 모듈 구현
- 알림 메시지 Block Kit 구성:

```
📄 새 정책보고서가 생성되었습니다

제목: 모빌리티 이용실태 분석 보고서
응답자: respondent-001
생성일: 2026년 2월 18일

[📥 DOCX 다운로드]  [👁️ 미리보기]
```

- DOCX 생성 완료 후 슬랙 알림 자동 트리거

#### 5-3 실습: Vercel/Render 배포 (실습실, 1h)

- GitHub 연동 자동 배포 파이프라인 구성
- 환경 변수 설정: `OPENAI_API_KEY`, `MONGODB_URI`, `SLACK_WEBHOOK_URL`
- 배포 URL로 실제 보고서 생성 → DOCX 다운로드 → 슬랙 알림 수신 데모
- `vercel --prod` 한 줄 배포 실습

#### 5-4 실습: 종합 프로젝트 (실습실, 1h)

수강생이 1~3번 과정에서 수집·분석한 모빌리티 데이터를 직접 투입하여:

1. 설문/데이터 입력 → 보고서 JSON 자동 생성
2. 차트(QuickChart) + 커버 이미지(DALL-E) 자동 삽입
3. DOCX 다운로드 확인
4. 슬랙 채널 알림 수신 확인
5. 발표 (배포된 URL 공유)

## 기술 스택 (4·5번 실습 기준)

```
Cursor (바이브코딩 IDE)
├── Next.js 14 App Router     ← 보고서 생성 웹 애플리케이션
├── TypeScript                ← 타입 안전 코드
├── MongoDB (Mongoose)        ← 세션·응답 데이터 저장
│
├── OpenAI GPT-4o-mini        ← 보고서 텍스트 자동 생성
├── OpenAI DALL-E 3           ← 커버 이미지 AI 생성
├── QuickChart.io             ← 차트 이미지 (외부 API, 설치 불필요)
├── docx (npm)                ← Word 파일 생성
│
├── Slack Incoming Webhook    ← 보고서 완성 자동 알림
└── Vercel / Render           ← 클라우드 배포
```
